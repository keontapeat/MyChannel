Architecture on Google Cloud

  • Delivery & Storage: Cloud CDN + Cloud Storage (hot/cold media tiers)
  • Encoding: Transcoder API for multi‑bitrate adaptive streaming
  • Compute: Cloud Run (initial) → GKE (scale) for microservices
  • Messaging: Pub/Sub for ingest, processing, fan‑out
  • Data: Firestore (operational) + Cloud SQL (relational)
  • Analytics & AI: BigQuery + Vertex AI for recommendations
  • Edge & Secrets: Cloud Armor, Secret Manager, Cloud Monitoring
flow: client ↔ CDN ↔ (Run/GKE APIs) ↔ Storage/SQL/Firestore • jobs via Pub/Sub → Transcoder • insights via BigQuery → recsys via Vertex

Security & Privacy

  • AES‑256 at rest (KMS), TLS 1.2+ in transit
  • Least‑privilege IAM, per‑service identities, audit logging
  • DDoS/WAF protection with Cloud Armor, rate limits
  • Observability: Cloud Monitoring, SLOs/SLIs, budgets/alerts
  • Privacy: GDPR/CCPA‑ready data handling and retention
Security by Design

Credit Utilization Plan (18 mo)

  • Phase 0 (0–2 mo): MVP on Cloud Run, Storage, Transcoder; CI/CD; monitoring & budgets.
  • Phase 1 (3–6 mo): CDN rollout, Pub/Sub pipelines, auth hardening, load tests, regional HA.
  • Phase 2 (6–12 mo): Multi‑region failover, BigQuery analytics, Vertex recsys, FinOps optimization.
  • Phase 3 (12–18 mo): SLA 99.95%+, autoscaling tuning, privacy reviews, public status.

Guardrails: budgets, anomaly alerts, perf/load gates, staged rollouts.
